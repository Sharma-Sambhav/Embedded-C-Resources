Bit Manipulation

Every peripheral register is a packed collection of bits. You must set, clear, toggle, and read individual bits without disturbing the others. Four operations cover 95% of firmware needs.

Interactive register — click bits to toggle
8-bit register (click any bit)
The four operations
bitops.c
uint8_t reg = 0b00101100; // SET bit n → OR with mask reg |= (1 << 3); // CLEAR bit n → AND with inverted mask reg &= ~(1 << 5); // TOGGLE bit n → XOR with mask reg ^= (1 << 2); // CHECK bit n → AND, result is non-zero if set if (reg & (1 << 4)) { handle_event(); } // Multi-bit field: set CR1[5:4] = 0b10 reg &= ~(0x3 << 4); // clear the two bits first reg |= (0x2 << 4); // then write the value // STM32: enable GPIOA clock (real code) RCC->AHB1ENR |= RCC_AHB1ENR_GPIOAEN;

| (OR)

Sets bits. Never clears. Use to turn bits ON.

& (AND)

Keeps bits. Use with ~mask to clear specific bits.

^ (XOR)

Flips bits. Toggle a pin or invert a flag.

~ (NOT)

Inverts all bits. Essential for building clear masks.

volatile

Tells the compiler: "this value can change outside your knowledge — never cache it in a register, never optimise away a read or write." Missing volatile causes bugs that only appear in release builds, with optimisation enabled.

The problem it solves
volatile_demo.c
// WITHOUT volatile — BROKEN at -O2 uint32_t *STATUS = (uint32_t*)0x40020010; while (*STATUS == 0) { } // Compiler sees: STATUS never written in this loop // Optimises to: while(1) {} ← infinite loop in hardware! // WITH volatile — CORRECT volatile uint32_t *STATUS = (volatile uint32_t*)0x40020010; while (*STATUS == 0) { } // Compiler: must re-read from address every iteration // ISR shared variable pattern volatile uint8_t rx_ready = 0; volatile uint8_t rx_byte = 0; void UART_IRQHandler(void) { rx_byte = UART->DR; // read hardware rx_ready = 1; // signal main — must be volatile! } void main_loop(void) { while (!rx_ready) { } // polls volatile — always re-reads process(rx_byte); rx_ready = 0; }

Hardware registers

Changed by hardware, not your CPU code. Re-read every access.

ISR shared flags

Set in interrupt, read in main. Without volatile, main sees stale cached value.

RTOS shared state

Variables shared across tasks. Prevents reordering across context switches.

volatile prevents caching — it does NOT guarantee atomicity on multi-byte variables. A 64-bit or struct update can still be interrupted mid-write. Use a critical section (__disable_irq()) for that.

Structs

Structs group related data — and in firmware they do something crucial: they map directly onto peripheral register blocks, giving you named access to every register at a known offset.

Peripheral register map pattern
gpio_regmap.c
typedef struct { volatile uint32_t MODER; // offset 0x00 — pin direction volatile uint32_t OTYPER; // offset 0x04 — output type volatile uint32_t OSPEEDR; // offset 0x08 — speed volatile uint32_t PUPDR; // offset 0x0C — pull-up/down volatile uint32_t IDR; // offset 0x10 — input data volatile uint32_t ODR; // offset 0x14 — output data volatile uint32_t BSRR; // offset 0x18 — bit set/reset } GPIO_TypeDef; #define GPIOA ((GPIO_TypeDef *)0x40020000) GPIOA->MODER |= (0b01 << 10); // pin 5 = output GPIOA->BSRR = (1 << 5); // drive pin 5 HIGH
Padding & alignment
struct { uint8_t a; uint16_t b; uint8_t c; } — compiler-inserted padding
+0a (1B) pad b_lob_hi c (1B) pad pad
sizeof = 8 bytes (not 4). Use __attribute__((packed)) only for protocol frames — it hurts performance.

-> operator

Dereferences pointer AND accesses member: GPIOA->ODR

Bitfields

uint32_t en : 1; packs named flags into a word.

Designated init

.field = val initialises specific members safely.

Memory Layout

Embedded systems have a fixed, non-virtual address space. Every address means something specific — Flash, SRAM, peripheral registers. Knowing the layout is essential for linker scripts, startup code, and debugging.

Cortex-M address map
0xFFFF_FFFFSystem / vendor peripherals (SysTick, NVIC, SCB)System
0x4000_0000Peripheral registers (GPIO, UART, SPI, I2C, DMA…)Peripheral
0x2000_0000SRAM — .data, .bss, stack, heap at runtimeRAM
0x0800_0000Flash — .text (code), .rodata (const strings)Flash
0x0000_0000Vector table / boot ROM (remapped to Flash on reset)Vectors
Binary sections
sections.c
// .text — compiled code (Flash, execute-only) void blink(void) { ... } // .rodata — read-only constants (Flash) const char msg[] = "hello"; // .data — initialized globals (stored in Flash, copied to RAM at boot) uint32_t counter = 0xFF; // .bss — zero-initialized globals (RAM only, no Flash cost) uint8_t rx_buf[64]; // Check your binary sizes: // arm-none-eabi-size firmware.elf // text data bss dec // 4096 32 128 4256
The startup code (crt0/SystemInit) copies .data from Flash to RAM and zeroes .bss before main() runs. If you forget this in a custom startup, your global variables will be garbage.

Interrupts + Shared Variables

An interrupt pauses the CPU mid-instruction, runs your ISR, then resumes. Any variable shared between the ISR and main code is a potential race condition — you must handle it explicitly every time.

Correct pattern
isr_pattern.c
// Step 1: volatile on every shared variable volatile uint8_t adc_done = 0; volatile uint16_t adc_value = 0; // Step 2: ISR — short, fast, no blocking void ADC_IRQHandler(void) { adc_value = ADC1->DR; // read result adc_done = 1; // signal main ADC1->SR &= ~ADC_SR_EOC; // clear flag — ALWAYS do this! } // Step 3: main polls the flag while (1) { if (adc_done) { adc_done = 0; // clear BEFORE using value process(adc_value); } } // Step 4: critical section for multi-byte access __disable_irq(); uint64_t snap = big_counter; // 64-bit — not atomic! __enable_irq();

ISR rules

Never block. Never printf. Return in microseconds, not milliseconds.

Clear the flag

If the hardware interrupt flag stays set, the ISR fires again immediately on exit.

Atomicity

32-bit R/W is atomic on Cortex-M. 64-bit and structs are NOT — use critical section.

Priority

Higher-priority IRQs pre-empt lower ones. Design priority levels deliberately.

Register Programming

Peripherals are configured by writing specific bit patterns to memory-mapped registers. Every GPIO mode, UART baud rate, timer period, and DMA channel is controlled this way — directly, with no abstraction layer.

Interactive — GPIOA MODER (click bits)
MODER: 2 bits per pin — 00=Input, 01=Output, 10=Alternate, 11=Analog
0x00000000
Toggle bits to update the register value
Read-modify-write — the golden rule
rmw.c
// WRONG — blows away every other bit in the register GPIOA->MODER = (0b01 << 10); // CORRECT — read, modify the field, write back GPIOA->MODER &= ~(0x3 << 10); // clear bits [11:10] GPIOA->MODER |= (0x1 << 10); // set bits [11:10] = 01 // Full UART init at 115200 (STM32, PCLK=16 MHz) RCC->APB2ENR |= RCC_APB2ENR_USART1EN; // clock on RCC->AHB1ENR |= RCC_AHB1ENR_GPIOAEN; // GPIO clock GPIOA->MODER &= ~(0xF << 18); GPIOA->MODER |= (0xA << 18); // PA9/PA10 = AF GPIOA->AFR[1] |= (0x77 << 4); // AF7 = USART1 USART1->BRR = 139; // 16M/115200 USART1->CR1 = USART_CR1_TE | USART_CR1_RE | USART_CR1_UE; while (!(USART1->SR & USART_SR_TXE)) { } USART1->DR = 'H';
